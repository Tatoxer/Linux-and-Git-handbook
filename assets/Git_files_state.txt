Состояния файлов в Git

Untracked - это новый файл, который не был добавлен для отслеживания командой git add. Все остальные файлы являются tracked

Tracked - отслеживаемые файлы


С отслеживаемыми файлами всё сложнее. Начнём с того, что они тоже могут находиться в 3-х разных состояниях:

unmodified (неизменённое) - файлы, которые не изменялись с последнего коммита

modified (изменённое) - файлы, которые поменялись с последнего коммита. Если попробовать вызвать 'git commit', то файлы в состоянии 'modified' в него не попадут. Предварительно их нужно перевести в состояние 'staged' сделав 'git add'

staged (подготовлено к коммиту)



Сброс изменений файла:

'git reset path/to/file' переводит файл из состояния 'staged' в 'modified'

'git checkout -- path/to/file' переводит файл из состояния 'modified' в 'unmodified', то есть по сути эта команда сбрасывает изменения.
